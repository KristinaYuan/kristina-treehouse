# Kristina's Treehouse — Design Specification

## Overview

Build a personal website deployed on GitHub Pages (with the possibility of migrating to a custom domain later).

The website is **not simply a blog**, but a long-term personal archive that combines:

* knowledge base
* reading notes
* projects
* stories
* essays
* development logs
* small moments in life

The overall feeling should be:

> **A quiet place worth coming back to.**

---

# Core Concept

The website is called

# Kristina's Treehouse

It is **a little library in the trees**.

Not a fantasy treehouse.

Not a children's website.

Instead, imagine a modern, peaceful treehouse overlooking the sky, containing books, notes, stories and ideas.

The treehouse is an underlying metaphor rather than a visual theme.

Avoid obvious tree illustrations, wooden textures or forest decorations.

The "treehouse" should mainly exist through wording, atmosphere and overall experience.

---

# Design Philosophy

Keywords:

* minimal
* elegant
* calm
* breathable
* reading-oriented
* timeless

Reference style:

* Claude
* Linear
* Notion

Lots of whitespace.

Very little decoration.

Content should always be the visual focus.

---

# Color Palette

Use a light blue theme rather than green.

The feeling should resemble:

* clear sky
* soft daylight
* quiet mornings
* fresh air

Suggested colors:

Background

```
#FAFCFF
```

Primary text

```
#2F3745
```

Secondary text

```
#667085
```

Divider

```
#E8EEF6
```

Accent color

```
#7AAEDB
```

Hover accent

```
#5D94D3
```

Avoid strong contrast.

Everything should feel soft and comfortable.

---

# Typography

Title:

Elegant serif font
(e.g. Cormorant Garamond / Libre Baskerville)

Body:

Clean sans-serif
(e.g. Inter)

Code:

JetBrains Mono

Typography should prioritize readability.

---

# Layout

Maximum content width:

Approximately 800px for articles.

Home page may be slightly wider.

Large whitespace.

No heavy cards.

No shadows.

No neumorphism.

Minimal borders.

Subtle dividers.

---

Layout:

Left:

Kristina's Treehouse

Right:

Knowledge
Reading
Ideas
Stories
Thoughts
Interlude
Notes
Archive

All navigation items should stay on the same line on desktop.

The first letters secretly spell

KRISTINA

This is an easter egg.

Do not explicitly mention it anywhere.

Users may discover it naturally.

# Homepage

The homepage is NOT a traditional blog homepage.

Instead, it resembles a GitHub profile combined with a personal library.

Suggested structure:

1. Website title

Kristina's Treehouse

Subtitle:

A little library in the trees.

(or another poetic sentence)

---

2. Short introduction

Very brief.

Just a few lines introducing the website.

---

3. Recent Activity

Inspired by GitHub Activity.

Show recent updates from all sections.

Examples:

* Added CSAPP Notes
* Updated Time Capsule
* Wrote Dream #23
* Finished reading XXX
* Published Thoughts: ...

Everything is displayed chronologically.

This replaces a traditional blog timeline.

---

4. Explore

Display the eight sections.

Knowledge

Reading

Ideas

Stories

Thoughts

Interlude

Notes

Archive

---

5. Optional

Currently Reading

Currently Building

Currently Thinking

Recently Listening

These provide a snapshot of current interests.

# Interaction

Animations should be subtle.

Only:

* fade
* opacity
* gentle underline
* soft color transition

No bouncing.

No scaling.

No flashy effects.

---

# UI Details

Sticky header.

Soft translucent background when scrolling.

Light blur effect.

Simple hover states.

No rounded colorful buttons.

No oversized icons.

Keep everything quiet.

---

# Overall Feeling

This website should feel like:

* sitting beside a window
* reading in a quiet treehouse
* looking at the sky
* writing thoughts that are worth keeping

The user should want to stay.

The interface should quietly disappear, allowing the content to speak.
